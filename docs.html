Troublemaker

Everything is not always ok

/
Returns the default troublemaker info page or a response from any of the internal views if configured.
Current Root View: {{if .CurrentRootView}}/{{.CurrentRootView}}{{else}}default{{end}}
/info
Returns plain text diagnostic data including GOMAXPROCS, NumCPU, and Build Settings.
/logs
Returns the last log.size lines of application logs in plain text.
/status/{codes}
Returns a random HTTP status from the provided comma-separated list of status codes.
/cache/[preset][.ext]
Returns a page with preset cache headers. The Content-Type is automatically set based on the file extension (e.g., .html, .json, .txt).
PresetDescription
no-cacheSets Cache-Control: no-cache
no-storeSets Cache-Control: no-store
public-1hSets Cache-Control: public, max-age=3600
private-1hSets Cache-Control: private, max-age=3600
Example with extension: /cache/public-1h.json
/set-headers/[header]/[value]/...[.ext]
Dynamically sets response headers based on the request path. Multiple header/value pairs can be provided. The Content-Type is automatically set if a file extension is provided at the end of the path. Example: /set-headers/Cache-Control/max-age=60/X-Custom-Header/Hello.html
/slow
Streams a random emoji at a specified interval.
ParameterTypeDefaultDescription
durationDuration{{.DefaultSlowDuration}}How long to keep the stream open (e.g., 10s, 2m).
intervalDuration{{.DefaultSlowInterval}}Frequency of character output (e.g., 500ms, 1s).
/load/
Manage and generate system load. Only a single load generator can run at one time.
EndpointDescription
statusReturns current status and full schedule of the active load test.
abortAborts any currently running load test.
cpuStarts a predefined CPU load pattern.
memStarts a predefined memory load pattern.
combinedStarts a combined CPU and memory load pattern.
sineStarts a sine wave load pattern (CPU and memory 180° phase shifted).
randomGenerates a new random seed and redirects to /load/seed/[seed].
/load/seed/[seed] Starts a reproducible random load schedule based on the provided seed (unsigned 64-bit integer). The schedule is always displayed, even if a load generator is already active. The test is constrained by the following limits:
  • Max CPU: {{.RandomLoadMaxCPU}}%
  • Max RAM: {{.RandomLoadMaxRAM}}MB
  • Total Duration: {{.RandomLoadTotalDuration}}
ParameterTypeDefaultDescription
durationDuration{{.RandomLoadTotalDuration}}How long the load test should run (e.g., 30m, 2h).
typesStringcpu,memComma-separated list of load types to include (cpu, mem).
The response contains the seed used and the full schedule of load steps.
/nothing
Opens a TCP connection but sends nothing (no headers, no body).
ParameterTypeDefaultDescription
durationDuration{{.DefaultNothingDuration}}Time to stall before hijacking and closing the connection.
/killhttp
Gracefully shuts down the HTTP server. The application will continue running but will no longer respond to HTTP requests.
/exit/[code]
Immediately terminates the troublemaker server process.
ParameterTypeDefaultDescription
codeInteger{{.DefaultExitCode}}The OS exit status code (0-127).
